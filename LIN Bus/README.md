Through reverse engineering of LIN bus communication across controlled scenarios, we demonstrate that EMI-spoofed sensor output is byte-identical to legitimate kick events and bypasses contextual validation, thereby triggering unintended state transitions during the locking sequence.

This folder includes data we exported from LIN bus connected to the kick sensing module in the tested Audi Q5, and the corresponding waveform captured from a oscilloscope. We obtained the data by tapping the LIN bus connected to the kick sensing module using probe-attached jumper wires and capture traffic under four controlled scenarios that combine two variables: the stimulus source (genuine kick vs. EMI injection) and the key fob state (within valid range vs. absent). In the EMI injection operations, we used a commercially available, portable broadband electromagnetic pulse generator, positioned 15–20 cm beneath the rear bumper of the Audi Q5, to observe the full interaction between the kick sensor and the liftgate controller. We turned the signal generator on and turn it off after 200ms-1s to mimic the temporal signature of a ral kick gesture, in which a leg enters and then esits the detection zone.

Among the active IDs on the bus, two are critical: ID 0x2F and ID 0x2E. For easier observation of the kick sensor response, we recommend filtering the captured data to show only these two frame IDs. By comparing the four scenarios, we establish the following findings.

Finding 1: Frame and byte role identification. We discover that ID 0x2F encodes the sensor detection state, identical to the bench-level findings, while ID 0x2E carries the liftgate control state that is gated by key fob authentication; byte [2] of ID 0x2F follows a consistent three-phase transition across all four scenarios: 0x00→0x01→0x03→0x00. The duration of each frame in this transition is consistent with the expected behavior of a capacitive kick sensor: detecting a change in proximity as the leg approaches (0x01, transiently), then confirming the gesture once the leg withdraws (0x03, longer). Finding 2: EMI-induced and genuine kick frames are byteidentical in data content. We compare the ID 0x2F frames generated under EMI injection with those generated by a physical kick gesture across all eight data bytes. The data content is byteidentical: both sources produce the same state transition sequence (0x00 → 0x01 → 0x03 → 0x00) with the same byte values in all fields, though inter-phase timing differs slightly owing to different stimulus sources. Critically, the liftgate controller responds identically in both cases: the same state transitions in ID 0x2E are triggered, and the same physical liftgate actuation occurs. This confirms that EMI coupling produces protocol-compliant frames upstream of the sensor module’s digital processing and LIN transceiver. The liftgate controller, and by extension the vehicle’s body control module, cannot distinguish between a genuine kick and an EMIspoofed event based on the LIN bus data, as no signal provenance verification exists at the protocol level. This is the root cause that enables our attack.
